OS Homework1 報告主題：Page Table

B10415030 四資工三 黃智威

前言

由於所有執行中的程式和資料都會放在memory中，佔用的空間都不相同，因此就會造成系統中，所有可用空間總和大於某個 process 所需要，但因為這些空間不連續所以無法配給該 process 使用，因此須靠用Page Table來管理virtual memory和physical memory，讓這些零碎空間可以被使用。

介紹名詞

physical address

一個page是4kB，因此offset是12bit(2^12=4kB)，所以有32-12位的PPN（physical page number）。

20bits 12bits
PPN Offset

virtual address

前10bits用來搜尋Page Directory，中間10bits用來搜尋Page Table，最後12bits因為內容一樣因此大小必須相同

10bits 10bits 12bits
Dir Table Offset

Page Directory

每個process都有自己的Page Table，因此必須再用Page Directory來存放所有Page Table。

運作原理

Page Table Entry中除了有PPN外還存放了flags，用來說明對應virtual address的使用權限，PTE_P表示PTE是否在Page Table中，如果不是就不允許使用。PTE_W控制能否寫入此page。PTE_U控制用戶process能否使用，不能的話只有內核可使用。

當OS拿到virtual address之後先從前面10bits找到對應process所在的Page Table，再利用接下來的10bit找到對應的physical page number，這就是physical address的前20bits，再將原本的virtual address最後12bits放入後就完成了。

static struct kmap {
  void *virt;       //存放virtual address 啟使位置
  uint phys_start;  //存放physical address 啟使位置
  uint phys_end;    //存放physical address 結束位置
  int perm;         //放置page table的flag
} kmap[] = {
 { (void*)KERNBASE, 0,             EXTMEM,    PTE_W}, // I/O space
 { (void*)KERNLINK, V2P(KERNLINK), V2P(data), 0},     // kern text+rodata
 { (void*)data,     V2P(data),     PHYSTOP,   PTE_W}, // kern data+memory
 { (void*)DEVSPACE, DEVSPACE,      0,         PTE_W}, // more devices
};

程式流程

代碼解釋

function1:kvmalloc(void)

用來呼叫兩個function，再來大多數工作由setupkvm來做







void
kvmalloc(void)
{
  kpgdir = setupkvm();
  switchkvm();
}

function2:setupkvm(void)

此副程式目的是要創建內核所需要的page table。一開始分配一個page來存放Page Directory，而k存放的是所須的內核的phycial address和virtual address位置，之後依續調用mappages來完成建立。




















pde_t*
setupkvm(void)
{
  pde_t *pgdir;
  struct kmap *k;

  if((pgdir = (pde_t*)kalloc()) == 0)
    return 0;
  memset(pgdir, 0, PGSIZE);
  if (P2V(PHYSTOP) > (void*)DEVSPACE)
    panic("PHYSTOP too high");
  for(k = kmap; k < &kmap[NELEM(kmap)]; k++)
    if(mappages(pgdir, k->virt, k->phys_end - k->phys_start,
                (uint)k->phys_start, k->perm) < 0) {
      freevm(pgdir);
      return 0;
    }
  return pgdir;
}

function3:mappages(pde_t *pgdir, void *va, uint size, uint pa, int perm)

主要工作是建立一個對應virtual address到phycial address的page table。一開始建立a和last來當做virtual address啟始跟結束，for迴圈中呼叫walkpgdir這個副程式把我們要的PTE找出來，之後初始化該PTE，再將a加上4096bytes，重新loop，最後跑到a==last時跳出迴圈






















static int
mappages(pde_t *pgdir, void *va, uint size, uint pa, int perm)
{
  char *a, *last;
  pte_t *pte;

  a = (char*)PGROUNDDOWN((uint)va);
  last = (char*)PGROUNDDOWN(((uint)va) + size - 1);
  for(;;){
    if((pte = walkpgdir(pgdir, a, 1)) == 0)
      return -1;
    if(*pte & PTE_P) 
      panic("remap");
    *pte = pa | perm | PTE_P;
    if(a == last)
      break;
    a += PGSIZE;
    pa += PGSIZE;
  }
  return 0;
}

function4:walkpgdir(pde_t *pgdir, const void *va, int alloc)

此副程式會找到virtual address所對應的page table並將值回傳給mappages，一開始用前10bits找到對應的page directory entry，假如不存在的話，代表對應的page table不存在，如果存在就再用接下來10bit得到PTE。






















static pte_t *
walkpgdir(pde_t *pgdir, const void *va, int alloc)
{
  pde_t *pde;
  pte_t *pgtab;

  pde = &pgdir[PDX(va)];
  if(*pde & PTE_P){
    pgtab = (pte_t*)P2V(PTE_ADDR(*pde));
  } else {
    if(!alloc || (pgtab = (pte_t*)kalloc()) == 0)
      return 0;
    // Make sure all those PTE_P bits are zero.
    memset(pgtab, 0, PGSIZE);
    // The permissions here are overly generous, but they can
    // be further restricted by the permissions in the page table
    // entries, if necessary.
    *pde = V2P(pgtab) | PTE_P | PTE_W | PTE_U;
  }
  return &pgtab[PTX(va)];
}

struct run {
  struct run *next;
};

//kmem是用來存放free list的，並且加入lock來管理
struct {
  struct spinlock lock; //放入lock 的struct
  int use_lock;         //選擇是否要用lock，要的話填1
  struct run *freelist; //放入free list
} kmem;

程式流程

代碼解釋

function1:kinit1(void *vstart, void *vend) & kinit2(void *vstart, void *vend)

分成兩個是因為啟動時只有4MB的記憶體需要使用kinit1來初始，但kinit1不能Lock因為Lock在4MB之外，所以kmem.use_lock要設0，等到建立好完整的page table，執行kinit2就可使用Lock因來保護free list，因此kmem.use_lock改設為1，再來就交給freerange處理。
















void
kinit1(void *vstart, void *vend)
{
  initlock(&kmem.lock, "kmem");
  kmem.use_lock = 0;
  freerange(vstart, vend);
}

void
kinit2(void *vstart, void *vend)
{
  freerange(vstart, vend);
  kmem.use_lock = 1;
}

function2:freerange(void *vstart, void *vend)

主要目的是將每個page交由kfree進行釋放和管理，利用PGROUNDUP函式可以確保此address是正規的，之後就循續的傳入kfree函式。









void
freerange(void *vstart, void *vend)
{
  char *p;
  p = (char*)PGROUNDUP((uint)vstart);
  for(; p + PGSIZE <= (char*)vend; p += PGSIZE)
    kfree(p);
}

function3:kfree(char *v)

此副程式主要將page的資料都清掉並且將page串到freelist中，清掉的方式是使用memset將整個4k的page都設成1，目的是假如有process錯誤隱用可以更快發現。設定完之後就可linklist到kmem中。




















void
kfree(char *v)
{
  struct run *r;

  if((uint)v % PGSIZE || v < end || V2P(v) >= PHYSTOP)
    panic("kfree");

  // Fill with junk to catch dangling refs.
  memset(v, 1, PGSIZE);

  if(kmem.use_lock)
    acquire(&kmem.lock);
  r = (struct run*)v;
  r->next = kmem.freelist;
  kmem.freelist = r;
  if(kmem.use_lock)
    release(&kmem.lock);
}
